# DebateAI 改进计划

> 基于代码审查，2026-05-14

## 一、用户交互核心问题

### 1. 🔴 辩论过程不可见（最关键）
- **问题**: 流式输出代码存在（SSE），但用户看不到实时流式文字。消息在 `append()` 中更新 `msg.content`，但 Alpine.js 的响应式渲染在高频更新下可能不流畅，且没有打字机效果。
- **表现**: 点击"开始辩论"后，面板长时间空白，然后消息突然整块出现。
- **方案**: 
  - 添加逐字流式渲染，带光标闪烁效果
  - 每个角色发言时显示"正在输入..."动画
  - 添加实时 token 计数器在输入区域

### 2. 🔴 用户无法参与辩论
- **问题**: 用户是纯旁观者。无法在辩论中途提问、引导方向、或注入评论。
- **方案**: 
  - 添加底部聊天输入框，允许用户在辩论中插入评论
  - 用户评论会被注入到下一轮的上下文中
  - 支持 @角色 定向提问

### 3. 🟡 三面板布局混乱
- **问题**: 
  - 左=建设方(橙色)，中=审查方(琥珀色)，右=侧边栏
  - 建设方和审查方的颜色太接近（橙 vs 琥珀），难以区分
  - 中间面板底部有"停止"和"导出"按钮，但左面板底部没有，逻辑不对称
- **方案**: 
  - 统一按钮位置，左右面板底部都有操作按钮
  - 增强颜色对比度（如蓝/橙 或 绿/红）
  - 考虑合并为双面板+底部输入的布局

### 4. 🟡 设置体验差
- **问题**: 
  - 新用户打开后看到空界面，不知道要先配置模型
  - 设置抽屉内容过多（模型管理+轮次+角色+Prompt），缺乏引导
  - 模型测试按钮需要填写3个字段后才能用
- **方案**: 
  - 首次使用时显示引导浮层
  - 设置面板分步引导（Step 1: 配模型 → Step 2: 选角色 → Step 3: 开始）
  - 添加预设 API 配置（DeepSeek、OpenAI 等一键填充）

### 5. 🟡 缺少键盘快捷键
- **问题**: 无 Cmd+Enter 开始辩论、Esc 关闭面板等快捷键
- **方案**: 添加常用快捷键

### 6. 🟢 移动端适配缺失
- **问题**: 固定布局，overflow:hidden，手机完全不可用
- **方案**: 添加响应式断点，移动端切换为单面板+Tab切换

### 7. 🟢 导出/分享不便
- **问题**: 只能下载文件，不能复制到剪贴板、分享链接
- **方案**: 添加"复制到剪贴板"按钮，支持生成分享链接

### 8. 🟢 辅助功能缺失
- **问题**: 无 ARIA 标签、无键盘导航、无高对比度模式
- **方案**: 添加基础 ARIA 属性和键盘导航

## 二、开发任务拆分

### Task 1: 流式输出可视化 + 辩论过程体验
**优先级**: P0
**改动范围**: `frontend/index.html` - CSS + JS + HTML
- 添加逐字打字机效果（CSS animation + JS 逐字追加）
- 添加光标闪烁效果（`█` 光标）
- 每个角色发言时显示 avatar + "正在输入..." 状态
- 流式过程中的实时 token 计数
- 添加消息流的时间线视图（Round 分隔线更清晰）

### Task 2: 用户聊天输入 + 辩论中互动
**优先级**: P0
**改动范围**: `frontend/index.html` - HTML + JS
- 在底部添加聊天输入框（类似 ChatGPT 的输入区）
- 支持在辩论过程中发送评论
- 用户评论注入到下一轮 context
- 支持 @角色名 定向提问
- 输入框支持 Cmd+Enter 发送

### Task 3: 布局优化 + 颜色区分 + 响应式
**优先级**: P1
**改动范围**: `frontend/index.html` - CSS + HTML
- 重新设计颜色方案：建设方=蓝色系，审查方=橙色系
- 统一左右面板的操作按钮
- 移动端响应式布局（单面板 + Tab）
- 改进空状态设计

### Task 4: 设置引导 + 预设配置 + 快捷键
**优先级**: P1
**改动范围**: `frontend/index.html` - HTML + JS
- 添加首次使用引导浮层
- 预设 API 配置（DeepSeek、OpenAI、Moonshot）
- 添加键盘快捷键（Cmd+Enter、Esc、Cmd+K 搜索）
- 设置面板分步骤折叠

### Task 5: 浏览器自动化测试
**优先级**: P2
**改动范围**: 新建测试文件
- 使用 Playwright/Puppeteer 自动化测试
- 测试：页面加载、配置模型、开始辩论、查看结果
- 测试：文件上传、历史记录、导出功能
